# 카풀 플랫폼 시나리오

## 1. 목적

- 출퇴근 수요를 기반으로 같은 방향의 이용자를 빠르게 연결한다.
- 차량 정보, 출발 시간, 탑승 위치, 도착 위치를 표준화해 운영 혼선을 줄인다.
- 매칭, 탑승, 이동, 도착까지의 흐름을 단순하고 명확하게 만든다.

## 2. 핵심 원칙

- 운전자와 탑승자는 출발 전에 필요한 정보를 모두 확인한다.
- 차량은 사전 등록된 정보로만 운영한다.
- 탑승 위치와 도착 위치는 구역 단위로 관리한다.
- 지연, 취소, 미탑승 같은 예외 상황은 즉시 처리할 수 있어야 한다.

## 3. 시나리오 A: 운전자가 동승자를 모집하는 경우

1. 운전자는 출발 전에 차량 정보와 이동 경로를 등록한다.
2. 탑승자가 신청하면 운전자는 탑승자의 픽업위치를 확인한 뒤 동승 가능 여부를 선택한다.
3. 매칭이 완료되면 탑승자에게 차량 번호, 차량 색깔, 차량 사진, 차종, 탑승 위치, 예정 출발 시간이 안내된다.
4. 운전자는 지정된 탑승 장소에서 대기한 뒤 출발한다.
5. 지연이나 경로 변경이 생기면 플랫폼이 즉시 알림을 보낸다.

## 4. 시나리오 B: 이용자가 카풀을 요청하는 경우

1. 이용자는 출발지, 도착지, 희망 시간을 입력한다.
2. 플랫폼은 근처에서 출발하는 차량을 검색한다.
3. 이용자는  예상 도착 시간, 경로 확인한다.
4. 조건이 맞는 차량을 선택하고 탑승을 요청한다.
5. 운전자가 요청을 수락하면 탑승 위치와 차량 정보가 확정된다.
6. 이용자는 정해진 장소에서 차량을 확인한 뒤 탑승한다.
7. 탑승 이후에는 실시간 위치와 도착 예정 시간이 공유된다.

## 5. 예외 처리

- 운전자가 늦는 경우: 자동 알림을 보내고 대체 차량을 제안한다.
- 이용자가 미탑승한 경우: 일정 시간 후 자동 취소 처리한다.
- 경로 변경이 필요한 경우: 운전자 승인 후 목적지와 ETA를 다시 계산한다.
- 안전 이슈가 발생한 경우: 즉시 신고하고 이용을 중단한다.

## 6. 운영 포인트

- 탑승 위치는 정확한 좌표기반으로 운전자의 차량 위치를 실시간으로 조회하고 수집한다 
- 운전자의 차량위치를 탑승자는 실시간으로 확인가능하다
- 출발 시간은 분 단위보다 5분 단위로 묶어 관리하면 매칭이 쉬워진다.
- 매칭 실패율이 높다면 출발 시간대와 경로 범위를 다시 조정한다.
